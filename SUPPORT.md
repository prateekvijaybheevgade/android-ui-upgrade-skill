## 🐛 Reporting Issues

### Before Reporting

1. **Check the FAQ**: [FAQ.md](./docs/FAQ.md) — Your question might already be answered
2. **Search existing issues**: [GitHub Issues](https://github.com/yourusername/android-ui-upgrade-skill/issues)
3. **Check TROUBLESHOOTING**: [docs/TROUBLESHOOTING.md](./docs/TROUBLESHOOTING.md)
4. **Search Discussions**: [GitHub Discussions](https://github.com/yourusername/android-ui-upgrade-skill/discussions)

### How to Report a Bug

1. Go to [New Issue](https://github.com/yourusername/android-ui-upgrade-skill/issues/new)
2. Choose "Bug Report" template
3. Fill in the template completely:
   - **Title**: Clear, concise bug description
   - **Steps to reproduce**: Step-by-step instructions
   - **Expected behavior**: What should happen
   - **Actual behavior**: What actually happened
   - **Environment**: OS, VS Code version, skill version
   - **Screenshots**: If applicable
   - **Error logs**: From Output panel

### How to Request a Feature

1. Go to [New Issue](https://github.com/yourusername/android-ui-upgrade-skill/issues/new)
2. Choose "Feature Request" template
3. Fill in the template:
   - **Title**: Clear feature description
   - **Problem**: What problem does this solve?
   - **Solution**: How should this work?
   - **Examples**: Real-world use cases
   - **Why it matters**: Impact on developers
